In this blog post, we'll recreate our own implementation of Python's range() function. There's a surprising amount of power and programming concepts behind this familiar function. The range() function is one of the first things beginners learn in Python. It's useful for for loops when you need to run a block of code a certain number of times:

for i in range(5):
    print('My name is Johnny Five Times.')

This will output:

My name is Johnny Five Times.
My name is Johnny Five Times.
My name is Johnny Five Times.
My name is Johnny Five Times.
My name is Johnny Five Times.

This for loop will also set i, the loop variable, to the values 0 to 4 (that is, up to but not including 5). You can provide two arguments to range() to give it a starting value besides 0:

for i in range(3, 6):
    print(i)

...and this code outputs:

3
4
5

You can also provide three arguments for a start, stop, and "step" value. The i variable ("i" is conventionally used because it is short and often the loop variable acts as an index into a data structure) will increase by two in this loop:

for i in range(3, 9, 2):
    print(i)

...and this code outputs:

3
5
7

But in Python, you can use several types of values with for loops. You can use lists, strings, or dict_items objects returned by the items() dictionary method:

for char in 'Hello':
    print(char)

for item in ['cats', 'dogs']:
    print(item)

for key, value in {'cat': 'Zophie', 'dog': 'Indiana'}.items():
    print(key, value)

...and this code outputs:

H
e
l
l
o
cats
dogs
cat Zophie
dog Indiana

What these values have in common is that their data types are "iterables". They are values that contain multiple values that can be presented to a for loop one at a time. This even applies to unordered data types like dictionaries. (Dictionaries contain multiple values, but there is no "first" or "last" value like lists have.)
